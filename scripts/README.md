# Chennai Metro Data Processing Scripts

This folder contains Python scripts for processing and generating data for the **Chennai Metro Intelligence Map** visualization project.

## Scripts Overview

### 1. `process_cmrl_data.py` (Recommended)
**Real Data Pipeline** â€” Processes Chennai Metro station and ridership data into visualization-ready JSON files.

- Loads station data from CSV files
- Generates realistic hourly, weekday, and weekend ridership profiles
- Creates OD (Origin-Destination) flow matrices
- Outputs: `stations.json`, `stations.geojson`, `ridership_hourly.json`, `ridership_weekday.json`, `ridership_weekend.json`, `od_flows.json`, `population_grid.json`

### 2. `process_data.py`
Simple utility that converts `stations_sample.csv` to `stations.json`.

### 3. `generate_placeholder_data.py`
Development helper â€” Generates all placeholder JSON data files for testing the UI without real data.

### 4. `fetch_metro_lines.py`
Downloads real Chennai Metro line geometries from OpenStreetMap and outputs `metro_lines.json` with actual route paths.
- Uses OpenStreetMap Overpass API
- Handles parallel track deduplication
- Generates simplified paths for efficient rendering

### 5. `requirements.txt`
Python dependencies for all scripts.

## Installation & Usage

### Step 1: Install Dependencies
```sh
pip install -r requirements.txt
```

### Step 2: Run the Data Pipeline

**Option A: Full Pipeline (Recommended)**
```sh
python process_cmrl_data.py
```
This processes all station and ridership data in one step.

**Option B: Step-by-Step**
```sh
# Process station data
python process_data.py

# Generate placeholder ridership data (if real data unavailable)
python generate_placeholder_data.py
```

### Step 3: Fetch Metro Line Geometry (Optional but Recommended)
```sh
python fetch_metro_lines.py
```
Downloads real metro line routes from OpenStreetMap. If API times out, the existing `metro_lines.json` will be preserved.

## Output Files

All scripts write to `../public/data/`:

| File | Purpose | Generated By |
|------|---------|-------------|
| `stations.json` | Station metadata (code, name, coordinates) | `process_cmrl_data.py` or `process_data.py` |
| `stations.geojson` | GeoJSON format for map rendering | `process_cmrl_data.py` |
| `ridership_hourly.json` | Hourly entry/exit by station | `process_cmrl_data.py` or `generate_placeholder_data.py` |
| `ridership_weekday.json` | Daily totals (weekday) | `process_cmrl_data.py` or `generate_placeholder_data.py` |
| `ridership_weekend.json` | Daily totals (weekend) | `process_cmrl_data.py` or `generate_placeholder_data.py` |
| `od_flows.json` | Origin-Destination matrices | `process_cmrl_data.py` or `generate_placeholder_data.py` |
| `population_grid.json` | Population heatmap grid | `process_cmrl_data.py` or `generate_placeholder_data.py` |
| `metro_lines.json` | Metro line paths with geometry | `fetch_metro_lines.py` |

## Data Sources

### Station Data
- **Primary**: `data/raw/stations_raw.csv` (if available)
- **Fallback**: `public/data/stations_sample.csv` (sample stations)

### Ridership Data
- Generated synthetically if real data unavailable
- Real data can be integrated from CSV/XLSX sources

### Metro Lines
- Fetched from **OpenStreetMap (OMS)** via Overpass API
- Uses official Chennai Metro OSM relation IDs
- Local fallback: `public/data/metro_lines.json`
