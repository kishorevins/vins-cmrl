
# Chennai Metro Intelligence Map

A modern, interactive visualization and analytics platform for the **Chennai Metro Rail Limited (CMRL)** system. Built with React, Vite, MapLibre GL JS, and Deck.gl for real-time ridership analysis, coverage mapping, and OD flow visualization.

**Live Demo**: http://localhost:5173 (after `npm run dev`)

## Features

- 🗺️ **Interactive Map**: Real-time metro line rendering with MapLibre GL JS
- 📊 **Ridership Analytics**: Hourly, weekday/weekend, and time-series analysis
- 🔄 **OD Flows**: Origin-destination flow visualization with Deck.gl
- 📍 **Station Lookup**: Search and inspect individual stations
- 🎨 **Dark Theme**: Optimized for data visualization with Tailwind CSS
- 📱 **Responsive Design**: Works on desktop, tablet, and mobile
- ⚡ **High Performance**: Optimized rendering with React hooks and Deck.gl

## Quick Start

### 1. Install Dependencies

```bash
# Frontend dependencies
npm install

# (Optional) Python dependencies for data scripts
cd scripts
pip install -r requirements.txt
cd ..
```

### 2. Generate Data

**Option A: Generate with Real Data (Recommended)**
```bash
cd scripts
python process_cmrl_data.py        # Generates all JSON files from station CSV
cd ..
```

**Option B: Use Placeholder Data**
```bash
cd scripts
python generate_placeholder_data.py  # Generates synthetic data
cd ..
```

**Option C: Fetch Metro Lines from OSM**
```bash
cd scripts
python fetch_metro_lines.py          # Downloads latest metro line geometry
cd ..
```

### 3. Start Development Server

```bash
npm run dev
```

Open http://localhost:5173 in your browser.

## Data Sources

### Stations
- **File**: `public/data/stations.json`
- **Source**: `public/data/stations_sample.csv`
- **Generated by**: `scripts/process_cmrl_data.py` or `scripts/process_data.py`

### Metro Lines
- **File**: `public/data/metro_lines.json`
- **Source**: OpenStreetMap (OSM) relations
- **Generated by**: `scripts/fetch_metro_lines.py`
- **Update**: Run `python fetch_metro_lines.py` to refresh

### Ridership Data
- **Files**: `ridership_hourly.json`, `ridership_weekday.json`, `ridership_weekend.json`
- **Generated by**: `scripts/process_cmrl_data.py` or `scripts/generate_placeholder_data.py`
- **To integrate real data**: Update CSVs and rerun data scripts

### OD Flows
- **File**: `public/data/od_flows.json`
- **Generated by**: `scripts/process_cmrl_data.py`

## Development Workflow

1. **Add/Modify Data**: Update CSV files in `public/data/` or `scripts/data/raw/`
2. **Regenerate JSON**: Run data processing scripts
   ```bash
   cd scripts
   python process_cmrl_data.py
   cd ..
   ```
3. **Test Frontend**: `npm run dev` — changes auto-reload
4. **Build for Production**: `npm run build`

## Frontend Architecture

### Key React Components
- **MapContainer.jsx**: Main map + Deck.gl visualization
- **LayerTabs.jsx**: Toggle visualization layers
- **StationPanel.jsx**: Station detail lookup
- **TimeSlider.jsx**: Temporal navigation
- **WeekdayToggle.jsx**: Weekday/weekend switcher
- **OdFlowControls.jsx**: Origin-destination flow parameters

### Custom Hooks
- `useMetroData()`: Load and cache all JSON data
- `useTimeSlider()`: Manage time state
- `useBreakpoint()`: Responsive design
- `useIsMobile()`: Mobile detection

### Visualization Layers (Deck.gl)
- **metroLinesLayer.js**: Metro line paths with arrows
- **volumeLayer.js**: Station ridership as circles
- **odFlowLayer.js**: Animated OD flow arcs
- **coverageGapLayer.js**: Station coverage analysis
- **weekdayWeekendLayer.js**: Weekday vs weekend comparison

## Troubleshooting

### "Could not load metro data" Error
- Run `python scripts/process_cmrl_data.py` to regenerate data files
- Ensure all JSON files exist in `public/data/`

### Map not rendering
- Check that `metro_lines.json` has correct structure with "path" property
- Verify coordinates are in [lon, lat] format
- Run `python scripts/fetch_metro_lines.py` to refresh metro lines

### Data not updating after script run
- Hard refresh browser: **Ctrl+Shift+R** (Windows) or **Cmd+Shift+R** (Mac)
- Clear browser cache

### Python script fails
- Install dependencies: `pip install -r scripts/requirements.txt`
- Ensure Python 3.8+
